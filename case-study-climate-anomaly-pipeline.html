Climate Data + ML Platform

California-Portugal Climate Pipeline

This case study demonstrates end-to-end platform engineering for data and ML workloads: ingestion, transformations, orchestration, tested features, and baseline model outputs.

Back to Portfolio

Problem

ML projects often stop at notebooks. Reliable systems require repeatable data ingestion, validated transformations, orchestration, and tested feature pipelines feeding model training.

Approach

  • Daily climate ingestion and normalization into DuckDB.
  • Layered dbt models (landing, clean, anomaly, ML features).
  • Prefect orchestration for daily and backfill runs.
  • Baseline RandomForest anomaly prediction with stored metrics.
  • Automated tests for transform logic, observability tables, and feature integrity.

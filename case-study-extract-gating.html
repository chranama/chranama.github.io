Flagship Case Study

Extract Capability Gating

LLM Extraction Platform

This case study shows how model capability onboarding and runtime policy enforcement prevent unsupported extraction behavior and produce reviewer-friendly evidence artifacts.

Back to Portfolio

Problem

Extraction endpoints can appear to work even when a model is not properly onboarded for schema-constrained output. This introduces reliability risk: invalid output structure, brittle behavior, and unclear operator confidence.

Approach

  • Offline onboarding/evaluation artifacts set extraction capability in patched model manifests.
  • Runtime policy checks capability before serving extraction requests.
  • Demo scripts execute PASS and FAIL flows and write deterministic evidence manifests.

Proof Artifacts

Claim: extraction behavior is governed by capability policy and verified with deterministic artifacts.

Command: python proof/validate_evidence_manifest.py

Artifacts:

  • proof/evidence_manifest.latest.json
  • traffic_out/phase41_20260304T230327Z/evidence_manifest.json
  • traffic_out/phase41_20260304T230327Z/host_pass_runtime.json
  • traffic_out/phase41_20260304T230327Z/host_fail_runtime.json

Signal: PASS runtime is extract-capable; FAIL runtime is capability-blocked.

Deployment extension: the same service now has a canonical Kubernetes proof path. Run python proof/generate_canonical_manifest.py to refresh proof/artifacts/phase5_k8s_kind/; the local kind deployment must pass smoke checks and the production overlay must render.

Async extension: the same system now supports queued extraction through a separate worker process. Run python proof/generate_canonical_manifest.py to refresh proof/artifacts/phase6_extract_async/; the canonical signal is 202 submission, separate worker execution, and a final succeeded job result.

5-10 Minute Reviewer Path

  1. Open the reviewer path.
  2. Run python proof/generate_canonical_manifest.py.
  3. Inspect PASS/FAIL gating artifacts in proof/evidence_manifest.latest.json.
  4. Inspect the Kubernetes proof in proof/artifacts/phase5_k8s_kind/.
  5. Inspect the async extract proof in proof/artifacts/phase6_extract_async/.
